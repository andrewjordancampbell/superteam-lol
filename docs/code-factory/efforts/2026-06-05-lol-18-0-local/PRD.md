# PRD: Worlds Run Local Prototype

## Context

Andrew wants an 82-0 style game for League of Legends esports, focused on winning Worlds with pro players using their real stats. The first milestone is a local prototype; deployment comes later after the gameplay loop is proven.

## Goals

- Build a local-first browser app for drafting one Top, Jungle, Mid, Bot, and Support.
- Use current pro esports stats, not fictional ratings.
- Show a clear roster projection that makes the goal feel concrete: win Worlds.
- Keep the app static-deployable so a later Vercel/Render/Netlify deployment is simple.

## Non-Goals

- No account system.
- No backend service.
- No live database.
- No Riot-authenticated features.
- No monetization or ads in this prototype.

## Product Surface

- Score band with result, score, title odds, and filled slots.
- One central spin panel that deals a team plus timeframe.
- Team logo treatment on the active roll.
- Stats/Blind toggle: Classic shows player stats; Blind hides the stat grid and ratings context.
- One-click player choice cards with player portraits for available roles from the rolled roster.
- Draft board with five locked role slots: Top, Jungle, Mid, Bot, Support.
- One reroll per run.

## Data Source

The prototype uses Oracle's Elixir aggregated player stats from `https://oe.datalisk.io`, generated by `scripts/build-pro-data.mjs` into `public/data/pros.json`. Visual assets come from hosted LoL Esports team/player image URLs when available.

The current local snapshot was generated on `2026-06-06T03:42:40.413Z` and contains 317 players across 12 tournament selections. Player entries preserve their actual team for each tournament/timeframe so the roll is truly team + timeframe. Asset coverage is 301 player portraits and 59 team logos.

## Deployment Shape

The app is a Vite React static build. A later deployment can serve `dist/` after running `npm run build`.
